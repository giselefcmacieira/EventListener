O que é um EventListener?
Um EventListener torna possível a execução de alguma ação ou uma função quando o usuário interage de uma determinada forma com a interface web.
É possível adicionar ou remover um EventListener utilizando os métodos:
  • - addEventListener()
  • - removeEventListener()
syntax:

[add ou remove]EventListener

(type, listener, options)
Parâmetros utilizados (type, listener, options)
  • - type
  • Representa o tipo de evento que o EventListener está esperando. Ou seja, o evento que irá causar a execução de uma função ou mais.
  • Existem muitos tipos de eventos que podem ser inseridos como parâmetro para o EventListener, os mais comuns e mais utilizados são os eventos de mouse e de teclado.
  • Alguns exemplos são:
    • - click
    • - dblclick
    • - mouseup
    • - mousedown
    • - keydown
    • - keypress
    • - keyup
  • - listener
  • Também conhecido como função de callback, o listener é o parâmetro que nos permite inser o que desejamos fazer assim que o evento que foi configurado no parâmetro type acontece.
  • Este parâmetro normalmente é uma função, mas também pode ser configurado como null e neste caso nada acontece.
  • Por padrão a função de callback (listener) recebe o objeto 'Event' que contém diversas informações sobre o evento.
  • - options
  • É um parâmetro opcional que vem configuado como false por padrão.
  • As opções possíveis são:
    - capture
    - once
    - passive
    - signal
